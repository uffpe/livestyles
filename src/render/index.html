The "Living Style Guide" is a document of all the general UI components used to build the responsive version of the Tabulex website. It uses the same code and structure as the live website, and acts as a common reference to all involved parties (designers, developers, Product Owners, support, clients etc.).

Purpose of this guide:

The main purpose is to make all in the team understand how the website should look and feel and act as a lexicon of building blocks for the developers to use.

If being used properly and being kept up-to-date it has the following advantages:

  • We can build consistently, focusing our energy on workflows and logic, not web forms and list items.
  • We can reuse code instead of reinventing the wheel.
  • We can see all of our UI components in one place, quickly revealing maintenance issues and test they work responsively and as intended accross different platforms and devices.

Who should use this guide:

  • Developers: Should use the guide as a lexicon of possible building blocks. The guide contains code examples and explanations of how to use the different components.
